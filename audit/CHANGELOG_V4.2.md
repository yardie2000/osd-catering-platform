# OSD Catering Platform — V4.2 Changelog

**Focus:** calculation correctness. The aggregation layer was already correct
(see [AUDIT_REPORT.md](AUDIT_REPORT.md)); V4.2 adds the two missing formula
stages, stops guessing the portion base where real data exists, and stops
treating "to taste" placeholders as orderable quantities.

## Calculation engine (the fix)

The shared scaler in `computeBatchOutputs` now implements the full kitchen formula:

```
Required   = recipeQty × (PAX / base)
Production = Required   × (1 + production_loss_pct/100)     ← NEW (was missing → E1)
Purchasing = Production ÷ (yield_pct/100)                   ← NEW (was forced = Production → E2)
```

- **`production_loss_pct` / `yield_pct`** are per-recipe overrides; `NULL` → global
  default (**10 % / 80 %**, `DEFAULT_CALC_CONFIG` in `lib/purchasing/aggregate.ts`).
  This is "Option 1: global defaults, per-recipe overridable".
- **Unit classification** (`classifyUnit`): loss/yield apply to **mass/volume only**.
  Counts (`Stück`/`stueck`, `Eigelb`/`eigelb`) scale by PAX but are not loss/yield-inflated;
  qualitative units (`Geschmack`, `Bedarf`, `EL`, `5-10Gr`) are passed through and flagged so
  the UI renders **"n. Bedarf"** instead of a nonsense number (→ E4). Matches the DB's
  normalised `unit_code` forms (ü→ue, space→_), verified against the live `units` table.
- **Single source of truth preserved:** production and purchasing still derive from
  one scaling pass and one config — they cannot diverge (verified by test).

### Yield data — surgical sync (NOT a full re-import)
A naive re-import of `OSD_Rezeptdatenbank_normalisiert_2_KORRIGIERT.xlsx` would **delete
all 68 menu→recipe links** (its `menu_items` sheet has no `recipe_code`; proven by
`audit/predict_import_links.cjs` → 0 links survive). Instead, `audit/yield_sync.sql`
(generated by `audit/gen_yield_sync.cjs`, strict "N Portionen" parse only) sets
`recipes.yield_quantity` for **11 recipes** — **applied to live DB**:
- batch-relevant: **Caesar SAU-010 200→45** (factor 0.9→4.0), **Zitrus SAU-011 150→175**.
- rollback: `audit/yield_rollback.sql`. Verified by `audit/verify_yield_sync.mjs` (11/11).
